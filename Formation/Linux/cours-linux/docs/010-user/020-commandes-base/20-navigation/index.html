    Navigation dans l'arborescence#

    Vue d'ensemble#

    Notion de chemin#

    Liste de déplacements séparés par des /

    • . = dossier en lui-même, dossier dans lequel on est
    • .. remonter dans l'arborescence = dossier parent
    • x => descendre dans le dossier x

    Il est d'usage d'indiquer

    • / à la fin du nom d'un dossier
    • rien pour un fichier

    Cette syntaxe est parfois utilisée par le shell (en plus des couleurs) pour faciliter la lecture

    Chemin absolu#

    • Déplacement à partir de la racine du systeme
    • Commencent TOUJOURS par un slash /
    • Le chemin absolu vers un fichier est toujours le même

    Chemins relatifs#

    • Déplacement à partir de votre dossier de travail
    • Ne commencent JAMAIS par un slash /
      • Mais le ./ de départ est optionnel

    Se localiser#

    La commande pwd (path of working directory = où suis-je en train de travailler)

    $ pwd
/home/batman/Documents

    Se déplacer#

    Changer de dossier de travail (change directory)

    cd DESTINATION

    Avec chemins absolus#

    $ cd /home/batman/documents
$ cd /usr/share/doc

    Avec chemins relatifs#

    $ cd ..
$ cd ../documents

    Cas particuliers#

    Revenir au répertoire précédent 🔙

    $ cd -

    Aller au répertoire de l'utilisateur ($HOME) 🏠

    $ cd ~/

    ou 

    $ cd

    Attention aux espaces#

    $ cd "nouveau repertoire avec des espaces"
$ cd nouveau\ repertoire\ avec\ des\ espaces

    Attention aux tirets#

    $ cd -- \-StupidFile

    Lister les fichiers d'un répertoire#

    La commande ls (list)

    ls [options] [objets]

    $ ls

    Liste avec aide sur la nature du fichier

    $ ls -f

    Liste détaillée 

    $ ls -l
$ ls -l /etc

    Liste complete (avec fichiers cachés)

    $ ls -a

    Liste recursive

    $ ls -R 
$ ls -R /var/www

    Note: on peut utiliser plusieurs de ces options en meme temps

    Nature des fichiers#

    C'est indiqué dans la réponse du ls -l

    b = périphérique de type bloc (possible de se ballader d'un endroit à l'autre d'un disque dur ou un autre périphérique). taille minimale définie par le systeme (ex 4096 octet), un bloc est alloué au fichier, qu'il soit entierement utilisé ou non.
c = périphérique de type caractère (lecture donnée une par une, comme via une paille) 
d = directory, dossier 
l = version lien, redirection, nom "raccourci"
- = fichier,

    Note.2: Unix n'a pas besoin des extensions de fichier pour savoir ce qu'il y a dans un fichier (voir la commande "file" qui permet de détecter la nature d'un fichier d'aprèss on contenu)

    Recherche de fichier#

    Commande : find DOSSIER

    Trouve les dossiers et fichiers

    attention, sur LINUX par défaut il le cherche dans le dosser courant, sur les vrais UNIX il faut rajouter "." après le find.

    On peut ajouter des filtres pour préciser la recherche !

    • -type f pour lister les fichiers
    • -type d pour les dossiers
    • -name 'quelquechose' pour trouver les noms contenant 'quelquechose'
      • utiliser le caractere générique * si on ne sais pas un bout du nom
      • ex : find . -name '*man'
    • -mindepth X OU -maxdepth OU -level : pour rechercher à un niveau précis

    On peut également chercher dans plusieurs dossier en meme temps 

    find DOSSIER1/ DOSSIER2/

    La commande retourne l'union des résultats pour chaque dossiers

    COMMANDE: whoami

    = qui suis-je ? (retourne le nom utilisateur) 🤓

    COMMANDE hostname

    retourne le nom du serveur

    Contenu des fichiers#

    Afficher le contenu d'un fichier#

    En entier

    $ cat ~/ToDo.txt

    ... seulement son début

    $ head ~/www/logs/access
$ head -n 20 ~/www/logs/error

    ... ou seulement sa fin

    $ tail ~/www/logs/access
$ tail -n 20 ~/www/logs/access
$ tail -f ~/www/logs/error

    Contenu des fichiers#

    Rechercher des fichiers#

    La commande grep

    $ grep 'Catwoman' ~/realVillains.txt
$ grep -r 'Catwoman' ~/villains/
$ egrep '(Catwoman|Poison Ivy)' ~/recentCrimes.txt

    La commande find

    $ find ~/documents/ -name \*.html
$ find ~/www/ -name \*.html -or -name \*.txt -exec grep -H {} \;

    Variables d'environnement#

    Alias de commandes (1/2)#

    Définition : alias nom-alias="longue commande à remplacer"

    Exemples de définitions :

    alias lls="ls -l"

alias filter_video="egrep -i '\.(avi|mov|mpe?g|mkv|wmv|rm)$'"
alias filter_audio="egrep -i '\.(wav|flac|mp3|aac|m4a|ogg|wma)$'"
alias filter_image="egrep -i '\.(jpe?g|png|xcf(.gz)|eps|svg|gif)$'"

    Alias de commandes (2/2)#

    Exemple d'usage:

    $ lls
...

    $ find |filter_video
